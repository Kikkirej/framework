# Architecture Documentation

Documentation can help...

* ... to onboard new team members
* ... to make decisions transparent
* ... communicate architecture to Stakeholders, team members, etc...
* ... sharing hints for specification of design and implementation
* ... clarity while creating the architecture
* ... base for architecture rating and reviews

## Seven rules for good documentation

As defined by "Softwarearchitekturen dokumentieren und kommunnizieren" from Stefan Zörner, Hanser Verlag

1. Write from the view of the reader
2. Avoid unnecessary Repetition
3. Avoid ambiguities
4. Use a default structure
5. document rationales for decisions
6. Keep it up to date, but not "too" up to date
7. Verify documentation for usability

## Quality attributes

* Currency
* Understanadbility 
* Maintainability
* Compactness
* Structure
