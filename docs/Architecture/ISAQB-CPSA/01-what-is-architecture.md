# What is architecture?

Not a clear definition, but many different definitions

For example:

**Kent Beck:**
> "Software architecture is what software architects do"
>
> — Kent Beck

**Eoin Woods:**
> "Software architecture is the set of design decisions which, if made incorrectly, may cause your project to be cancelled."
>
> — Eoin Woods

**IEEE 1471-2000:**
> "The fundamental organization of a system, embodied in its components, their relationships to each other and the environment, and the principles governing its design and evolution."
>
> — IEEE Std 1471-2000, "Recommended Practice for Architectural Description of Software-Intensive Systems"

Generally can be said: Architecture decisions are fundamental, and are hard to be changed over time. Consequences for bad decisions are: 

* higher cost
* time loss
* failing projects
