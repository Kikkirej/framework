# Compliance Assessment

> **Note:** This document was generated with Agentic AI.

## Role/Purpose
Deliverable that evaluates architecture deliverables against established architecture principles, standards, and regulations to ensure adherence and identify areas of non-compliance.

This document helps ensure that architectural decisions and implementations align with organizational policies, industry standards, and regulatory requirements.

## Contents

| Content | Description | Required |
|---------|-------------|----------|
| Assessment Scope | Define what is being assessed (architecture deliverables, implementations, projects) and the boundaries of the assessment | Yes |
| Compliance Criteria | List the principles, standards, regulations, and policies against which compliance is being evaluated | Yes |
| Assessment Methodology | Describe the approach and methods used to conduct the compliance assessment | Yes |
| Stakeholders | Identify key stakeholders involved in or affected by the compliance assessment | Yes |
| Assessment Findings | Document detailed findings for each compliance criterion, including evidence of compliance or non-compliance | Yes |
| Compliance Status Summary | Provide an overall compliance status with metrics (e.g., percentage compliant, number of violations) | Yes |
| Non-Compliance Issues | List all identified areas of non-compliance with severity ratings and impact assessment | Yes |
| Risk Assessment | Evaluate the risks associated with identified non-compliance issues | Recommended |
| Remediation Plan | Propose corrective actions and recommendations to address non-compliance issues | Recommended |
| Waivers and Exemptions | Document any approved waivers or exemptions from specific compliance requirements | No |
| Timeline and Milestones | Define timeline for remediation activities and follow-up assessments | No |
| Supporting Evidence | Reference supporting documentation, test results, or artifacts used in the assessment | Recommended |
| Approval and Sign-off | Record approval from relevant authorities (e.g., compliance officers, architecture board) | Yes |

## Document States

| State | Description |
|-------|-------------|
| Draft | Assessment is in progress and findings are being documented |
| Review | Assessment is complete and under review by stakeholders |
| Approved | Assessment is approved and findings are accepted |
| Remediation | Non-compliance issues are being addressed based on the assessment |
| Closed | All identified issues have been remediated or accepted |
| Obsolete | Assessment is no longer relevant or has been superseded by a new assessment |
